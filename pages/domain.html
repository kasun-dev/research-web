Research Domain

Project Domain & Literature

An overview of the research landscape, identified gaps, problem statement, objectives, methodology, and technologies used across all four subcomponents.

01

Literature Survey

Predictive Maintenance

Existing Solutions

State-of-the-art solutions like PredictiveOps assume stable connectivity and clean datasets. They fail in real-world industrial environments where sensor data is unreliable, incomplete, and context-poor.

AR in Industry

Current AR Limitations

Existing AR maintenance systems require specialized technical personnel to author guidance content. Automatic action recognition and spatial workflow authoring remain open research challenges.

Fault Diagnosis

Diagnostic Systems Today

Current systems either only predict failure without explaining root causes, or use static problem→solution mappings with no feedback loop or adaptability for new fault types.

Digital Twin

Full Twin vs. Data-Driven

True physics-based digital twins require detailed CAD and materials data. With only temperature, vibration, and noise sensors, a data-driven synthetic simulator is the realistic alternative.

02

Research Gap

IoT Pipeline Gap

Unreliable Raw Data

No pipeline bridges raw industrial IoT sensor data to ML-ready, context-rich datasets with fault-tolerance, redundancy, and intelligent preprocessing in one integrated solution.

AR Authoring Gap

Manual Workflow Creation

No automatic system exists that can recognize maintenance actions and generate AR overlay workflows without specialized authoring expertise.

Diagnostic Gap

No Guided Resolution

A diagnostic assistant capable of interpreting sensor abnormalities, identifying root causes, and delivering guided step-by-step resolution instructions does not yet exist in a unified system.

Predictive Gap

Black-Box Predictions

Existing predictive systems lack explainability, uncertainty quantification, and technician feedback loops — making them untrustworthy in production environments.

03

Research Problem

Industrial machinery maintenance at facilities like Haycarb PLC relies on manual inspection and reactive repair, leading to costly unplanned downtime. Existing predictive maintenance tools are siloed, untrustworthy black boxes that lack real-time AR guidance, intelligent fault diagnosis, and resilient IoT data pipelines — preventing technicians from acting on accurate, explainable predictions in the field.

04

Research Objectives

1

Fault-Tolerant IoT Pipeline

Design a resilient sensor pipeline using redundant hardware, dual-mode communication, and intelligent preprocessing to ensure >90% data availability for ML models.

2

Automatic AR Workflow Authoring

Build an action-recognition-based system that auto-generates AR maintenance guidance overlays without requiring specialized technical expertise to create them.

3

NLP-Driven Fault Diagnosis

Develop IFDRA — an NLP + LSTM assistant that interprets technician queries, classifies faults from sensor data, and delivers guided resolution steps with a continuous feedback loop.

4

Explainable Predictive Maintenance

Implement a data-driven virtual replica with XAI explanations, uncertainty-aware confidence scores, and technician feedback integration to continuously improve model accuracy.

05

Methodology

IoT Edge Layer

Data Acquisition

  • ESP32 NodeMCU gateway
  • Redundant sensors (ADXL345, MPU6050, DHT22, KY-037)
  • Cross-sensor verification for drift detection
  • Dual-mode: Wi-Fi General + GSM Emergency
Cloud Preprocessing

Intelligent Pipeline

  • Filtering: moving averages, Fourier transform
  • Timestamp synchronization & imputation
  • Feature engineering: statistical, spectral, trend
  • Metadata enrichment with trust scores
AR System

Authoring & Playback

  • Action recognition using HG14 / EGO4D datasets
  • Spatial anchors via Meta Quest 3S SDK
  • JSON DSL workflow storage
  • Unity + A-Frame rendering
AI Models

Diagnosis & Prediction

  • LSTM for sequential fault classification
  • Transformer NLP for symptom intent mapping
  • RandomForest → TCN/Transformer for prediction
  • XAI for sensor contribution explanations
